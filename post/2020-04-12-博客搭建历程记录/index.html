摘要

第一次尝试搭建个人博客，记录一下搭建的过程以及其中遇到的一些坑

平台以及generator的选择

其实一开始没准备自己搭，就是想记录一下最近学习日语罗马字的一些内容，准备在csdn写一写就算了（还能赚点积分）。最后想一想反正要搞不如自己搞一个，也不会受什么限制。本来自己也有服务器和域名，准备搞个wordpress的，但又不知道会不会不稳定，也比较麻烦以及说不定会有些其他问题，就想到了github.io，看了看知乎用的还挺多人用的就确定下了。

GitHub推荐的generator是Jekyll，一开始也整了挺长时间的换了好几个模板都不太满意，最后看到现在的模板挺好，下载以后才发现是Hugo的，就换到了Hugo上，而相比之下Hugo确实有不少优点。最明显的就是Hugo只需要一个二进制文件，而不像Jekyll是基于Ruby的，配置起来比较方便，而且国内用Ruby还需要用镜像。一开始配置的时候还遇到了一个兼容性的问题，装的是2.7.1的版本，用模板的时候提示

1

Error installing rails: nokogiri requires Ruby version < 2.7.dev, >= 2.3.

因为没经验习惯性地查了查于是换到了2.6版本的Ruby就ok了。

Hugo的搭建过程

初始化

首先是下载二进制文件，然后添加到环境变量PATH。我把一些零散的二进制文件都放到了~/apps里面，就不用每次都加环境变量了。By the way，我的PATH之前就爆了，新建了一个PATH_KAWAI来存，并且在PATH里面置顶了PATH_KAWAI

下载模板

然后是下载模板。考虑到国内的网络状况，我没有通过git clone，而是直接从GitHub网页下载的。虽然以后不能git pull来更新，但看了下模板最后的更新日期是2018年也就无所谓了。

调成参数并测试

然后就是用Hugo新建网站

1

hugo new site blogsite

拷贝模板

新建post

1

hugo new post/hello-world.md

编辑内容

调整config.toml的各种参数（因为我下的模板有部分中文注释所以非常容易）

最后运行

1

hugo server

就出现了一个问题，主页上不显示文章，而是显示名为Posts的目录。 一开始我尝试调各种参数也没有用，在bing上也搜索不到，最后Google搜到一篇文章说Hugo要用0.54版。确实，换了以后就好了，好气啊(╯‵□′)╯︵┻━┻善用搜索引擎的重要性

修改配色方案

按照模板的说明，在主题目录下的/src/css/_variable.scss里修改配色方案为Mint Green，然后依次执行

1
2

yarn install
yarn build

需要安装yarn和Node.js，因为之前想编译Hyper这个第三方terminal的时候都安装过直接都可以用，就是npm遇到了一点小问题，准备单独开一篇记一下。

静态网站的生成

在站点根目录执行

1

hugo

即可，最后我在GitHub客户端里直接push上去就完成啦

总结

总体上Hugo的使用还是非常容易的，只要在config.toml里配置好包括theme、baseURL等在内的各种参数，用到的命令都会非常好记（hugo hugo server hugo new *.md hugo new site *)而不用加什么参数。
