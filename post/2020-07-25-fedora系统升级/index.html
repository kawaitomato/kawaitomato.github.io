Fedora系统升级

前言

看到Fedora出了33，不禁有些手痒，开服务器时装的Fedora 31已经很久没有更新了，查了一下升级方法，看起来还挺容易就升了。参考文章给出了命令行和图形界面两种方法，由于使用ssh登录，只能用命令行了。

注：我是用root用户进行操作的，使用普通用户升级记得加上sudo。
注2：升级可能有一定的机率失败。我的服务器是自用的，崩了就崩了，如果是商用服务器记得备份。参照https://fedoramagazine.org/taking-smart-backups-duplicity/应该可以完成。

更新软件

1

dnf upgrade --refresh

安装dnf的系统升级插件

1

dnf install dnf-plugin-system-upgrade

开始升级

1

dnf system-upgrade download --releasever=33

安装了一会儿以后遇到了一个报错

1

Curl error (37): Couldn't read a file:// file for file:///etc/pki/rpm-gpg/RPM-GPG-KEY-fedora-33-x86_64 [Couldn't open file /etc/pki/rpm-gpg/RPM-GPG-KEY-fedora-33-x86_64]

查了查还挺常见的，解决方法如下：

1
2
3

cd /etc/pki/rpm-gpg
wget https://src.fedoraproject.org/rpms/fedora-repos/raw/master/f/RPM-GPG-KEY-fedora-33-primary
ln -s RPM-GPG-KEY-fedora-33-primary RPM-GPG-KEY-fedora-33-x86_64

大概是少了一个验证Fedora安装包的公钥，下载下来就好了。

之后再执行升级命令

1

dnf system-upgrade download --releasever=33

会要求验证公钥的身份

1
2
3
4
5

Importing GPG key 0x9570FF31:
 Userid     : "Fedora (33) <fedora-33-primary@fedoraproject.org>"
 Fingerprint: 963A 2BEB 0200 9608 FE67 EA42 49FD 7749 9570 FF31
 From       : /etc/pki/rpm-gpg/RPM-GPG-KEY-fedora-33-x86_64
Is this ok [y/N]:

这里我使用Kleopatra证书管理器在服务器上用uid查找这个证书的指纹（比较正统的做法可能是在Fedora官网上找，不过要麻烦点），对比后发现是一致的，就放心继续了。

重启

安装完成后，会有如下提示

1
2
3
4

Download complete! Use 'dnf system-upgrade reboot' to start the upgrade.
To remove cached metadata and transaction use 'dnf system-upgrade clean'
The downloaded packages were saved in cache until the next successful transaction.
You can remove cached packages by executing 'dnf clean packages'.

按照要求使用如下命令重启，即可完成安装

1

dnf system-upgrade reboot

验证

等待重启完成，使用ssh再次登录

1

ssh username@mydomain.com

可以通过uname命令查看系统版本，检查安装是否成功。看到fc33字样，就说明系统版本号是Fedora 33了

1

uname -a
