screen的使用

前言

可能是网络线路问题，ssh总是会中断。最近想编译一下MariaDB（系统升到了Fedora 33以后发现还没被MySQL支持，悲），要挺长时间的，需要一个不间断会话，这时候就需要screen啦。

screen是一款能够实现多窗口远程控制的开源服务程序，简单来说就是为了解决网络异常中断或为了同时控制多个远程终端窗口而设计的程序。用户还可以使用screen服务程序同时在多个远程会话中自由切换，能够做到实现如下功能。

会话恢复：即便网络中断，也可让会话随时恢复，确保用户不会失去对远程会话的控制。

多窗口：每个会话都是独立运行的，拥有各自独立的输入输出终端窗口，终端窗口内显示过的信息也将被分开隔离保存，以便下次使用时依然能看到之前的操作记录。

会话共享：当多个用户同时登录到远程服务器时，便可以使用会话共享功能让用户之间的输入输出信息共享。

注：系统Fedora 33，root用户

安装

1

dnf install screen

创建会话

手动创建

因为我是要编译程序，在这里给会话起名为make

1
2

screen -S make
make

这样就是在screen里完成编译了。

自动创建

1

screen make

这样不需要自己命名，就会在screen中完成make操作，在完成会会自动关闭这一screen

退出会话

在screen会话中执行

1

exit

即可退出。如果不在会话中，可以使用

1

screen -S make -X quit

查看会话

1

screen -ls

断开会话

使用Ctrl+A再按D的组合键，即可临时断开会话，这时候用screen -ls查看会话，会发现之前的状态(Attached)会变为(Detached)。

另外，在ssh中使用screen时断开ssh，原先的screen会话不会断开，而是保持Attached状态，这时候也需要手动断开

1

screen -d make

或者也可以在运行screen -ls命令后查看会话的id，一次作为标识断开会话，例如

1

screen -ls

这时会显示出

1
2
3

There are screens on:
    23849.make      (Attached)
1 Socket in /run/screen/S-root.

再执行

1

screen -d 23849

恢复会话

断开以后，原先会话内的工作还是在持续进行的，如果要恢复需使用

1

screen -r make

或者

1

screen -r 23849
